Biological interpretation of PDC protein expression data through heatmaps

The PDC processes the submitted raw mass spectrometry data through a common data analysis pipeline (CDAP) to produce derived analysis results which can be used to study identification of proteins and post-translational modifications (PTMs). On the PDC portal, these quantitative results can be visualized as a heatmap for downstream analyses of differential expression between samples at the protein- or PTM site-level.



Software and Access

The PDC includes an implementation of the client-side application, Morpheus, developed at the Broad Institute. The Mopheus website offers documentation, a tutorial, notes on configuration, and other resources. Morpheus on the PDC can be accessed primarily through the heatmap image links (see image at right) displayed on the study page (e.g., PDC000120), where each heatmap automatically loads the quantitative data for the selected study into the application.


Hierarchical clustering

The main purpose of heatmap visualization, in the context of the PDC, is to examine clustering of samples according to protein abundances. In a typical heatmap visualization, rows correspond to genes (or proteins) and columns correspond to samples (or aliquots.) Additional row (e.g., genes accessions) and column (e.g., clinical attributes) annotations can be embedded in the input file read by Morpheus.
When a study heatmap is loaded, the data have not been clustered by either row or column. Clustering can be performed according to the method of your choice by accessing the Tools menu. You will know your data have been clustered if above the column annotations is displayed a dendogram, linking the samples. In addition, the ordering of the columns will have changed according to the clustering calculations. Morpheus allows for a large variety of data analysis methods. If you are unfamiliar with the statistical methods available in the application, you may wish to visit the Morpheus site or consult with bioinformatics colleagues.


Data

Analysis results from the CPTAC CDAP (Common Data Analysis Pipeline) are the source data for heatmap visualizations on the PDC. These data are merged with various row and column annotations to generate a .gct file when the study is loaded into the PDC. For proteome studies, both log2 ratio and unshared_log2_ratio data are available for visualization. Unshared_log2_ratio data exclude data from shared peptides. Each cell represents the normalized log2 ratio from an iTRAQ or TMT experiment, where the numerator is the value for that gene/protein in that aliquot and the denominator is the expression level of whatever control channel was used in the study. Control channels are typically pooled samples from many tumor samples.
In addition to downloading publication-quality images from Morpheus, you may also export the data in .gct format for use with statistical analysis tools (e.g., in R or Python.) Exports are available from the File menu. We encourage you to try the available tools available from the Morpheus menu, including those under View->Options to help you identify potential correlations between annotation variables and the protein expression data.

What are the goals of PDC?

The primary goal of Proteomic Data Commons (PDC) is to democratize access to cancer-related proteomic datasets and also aims to provide sustainable computational support to the cancer research community through interoperability with other nodes, such as Genomic Data Commons and Cloud Resources, in the NCI Cancer Research Data Commons (CRDC) ecosystem.

Is PDC launched?

NCI PDC is officially launched on Mar 23, 2020. Click here to read the official announcement.

Do I have to login to PDC to access data?

No, there is no need to create an account or login to PDC to browse the portal or download the data. However login is required if you would like to submit data to PDC.

How do I submit data to PDC? How do I register my program with PDC?

Detailed instructions for submitting data into the PDC are available here.

How to find corresponding genomic and/or imaging data for the PDC proteomic data?

If a study in PDC has corresponding genomic and/or imaging data available in other resources, you may find the mapping of the individual cases to the external resources in the 'Clinical' tab on the browse page (see the image below). Either click on the thumbnail in the column 'Genomic or Imaging Data Resource' or you may also select the cases of interest and export the clinical manifest. The manifest file will have the resource name and identifier for each case.



What is the difference between PDC and CPTAC Data Portal?

The PDC will serve as a public data repository of mass spectrometry (MS) based proteomics data, and is maintained by the National Cancer Institute. PDC will host datasets from large consortiums such as Clinical Proteomic Tumor Analysis Consortium (CPTAC), International Cancer Proteogenome Consortium(ICPC) and Applied Proteogenomics OrganizationaL Learning and Outcomes (APOLLO) and also from independent research programs and grants.
The PDC Data Portal provides a platform for efficiently querying, visualizing, analyzing and downloading high quality, curated and harmonized proteomic datasets. All of the cancer cohort datasets from CPTAC data portal are already available on PDC. In the near future the CPTAC data portal will retired and all of the data will be made available through PDC.
PDC represents a state of the art repository technology and facilitates multiomic integration through interoperability with other nodes, such as Genomic Data Commons and Cloud Resources, in the NCI Cancer Research Data Commons (CRDC) ecosystem.
The data from the Clinical Proteomic Tumor Analysis Consortium is also available from the CPTAC Data Portal External Link.

How do I download data?

Individual files can be downloaded by clicking the 'Download' button in the File table.
To download multiple files, identify and select the files of your interest by clicking the check box. You may select multiple files at once by clicking the checkbox in the header row of the table. After the files are selected, click on the 'Download Files' button. Currently there is a limit on the number and volume of files that can be downloaded through this approach.
If you want to download multiple files programmatically, see 'How do I download multiple files at once?'.

How do I download multiple files at once?

Once you identify the files of your interest, move to 'Files' tab and select one or all files and then click 'Export File Manifest'. The manifest will contain a download URL each selected file in a separate row. The URLs will expire after 48 hours. You may revisit the PDC portal to generate a new file manifest.


On most linux, mac and cygwin environments with wget installed, you may provide the manifest file as input to the following code snippet to initiate batch download. 

                    #!/bin/bash

                    #This is a sample bash script to batch download PDC data files in to your local computer
                    #This works on most linux machines
                    #The script uses only the native shell functionalities and wget
                    #Save this script as downloadPDCData.sh in your local directory and set executable permissions
                    #Provide downloaded PDC File manifest as a command line parameter
                    #Ex:  ./downloadPDCData.sh -f <PDC File manifest>
                    #Users may improvise using awk, sed and rewrite in perl, python, etc.

                    helpFunction() {{ '{' }}
                        echo ""
                        echo  "Usage: $0 -f <Name of the manifest file downloaded from PDC in CSV format<"
                        exit  1 # Exit script after printing help
                    {{ '}' }}

                    while getopts ":f:" opt
                    do
                            case "$opt" in
                            f ) filename="$OPTARG" ;;
                            ? ) helpFunction ;; # Print helpFunction in case input file name is non-existent
                            esac
                    done

                    # Print helpFunction in case input filename is empty
                    if [ -z "$filename" ]
                    then
                        echo "Some or all of the parameters are empty";
                        helpFunction
                    fi

                    IFS="," #split the input csv file based on comma separator and read into an array

                    while read -r -a array
                    do
                        for index in "${{ '{' }}!array[@]{{ '}' }}"; do
                            #echo $index
                            #Extracting file URL from Column 11
                            if [[ "$index" == 11]; then
                                #echo "$index ${{ '{' }}array[$index]{{ '}' }}"
                                url="${{ '{' }}array[$index]{{ '}' }}"
                                temp="${{ '{' }}url%\"{{ '}' }}" #will remove the suffix " (escaped with a backslash to prevent shell interpretation).
                                temp="${{ '{' }}temp#\"{{ '}' }}" #will remove the prefix " (escaped with a backslash to prevent shell interpretation).
                                url=$temp
                                echo "Downloading $url........."
                                wget -c $url # -c allows resuming the failed or stopped downloads
                                                # -c also eliminates redownload if a file already exists (with same size) in the current directory.
                            fi
                        done
                    done > $filename

Why does the file download URLs throw an error or exception?

PDC is hosted on AWS cloud and is under active development. To reduce egress costs from unintended downloads, the URLs will expire after 48 hours. You may revisit the PDC portal to generate a new file manifest.

Where do I get more details about a study?

Click on the name of the study in the browse page, that opens up a study summary page. The study summary page provides details about the objective of the study, protocol, experimental design, clinical data of the cases and samples used in the study.

How can I find gene/s of my interest?

Genes can be searched with their gene symbols through the search box on PDC portal. Enter gene symbol or name of the gene (such as kinase) in the search box and click on the gene of your interest in the drop down. This will take you to a gene summary page.
It is also possible to search for multiple genes such as those involved in a pathway. Go to the Gene tab on the browse page and in the gene filter on the left hand side enter the list of gen symbols. You may also select from the prebuilt gene lists from several pathways important in cancer.

How do I explore protein quantitation data for a study through heatmaps?

The PDC Common Data Analysis Pipeline generates a protein abundance matrix rolled up to gene level for each study. The data can be viewed as an interactive heatmap through Morpheus viewer on PDC.
Morhpes is a heat maps viewer from Broad Institute. It is versatile and has a lot of features to filter, cluster and save the data.
Here is a quick tutorial to find 500 most variables genes by variance and perform hierarchical clustering.
Go to study summary page and click on the heatmap thumbnail. It opens a new page with the quantitation preloaded into morpheus viewer.
More ways to explore the heatmap can be found here accessed here - Morpheus - Tutorial External Link

What can I search on PDC?

PDC can be searched for the following:
- Biospecimens such as case, sample or aliquot using their original identifiers or PDC ids
- Studies using their PDC id (e.g. PDC000220) or partial name (e.g. CPTAC, HNSCC)
- Genes using their Gene symbol (e.g. BRCC3) or partial description (e.g. Kinase)
It is also possible to search for related data from external resources using the external identifiers. For example, you may search for a dbGaP study using its identifier (e.g., phs000892) to identify all related PDC studies.

Can I search for a particular peptide sequence within the datasets in PDC?

Peptides can be searched through PepQuery, a peptide-centric approach that focuses on only novel DNA or protein sequences of interest.
From the menu bar, go to Analysis -> PepQuery. Enter the peptide in the search box and select the dataset to search against. More details about PepQuery can be found here - https://www.ncbi.nlm.nih.gov/pmc/articles/PMC6396417/.

What type of files are available for download from PDC? What are the outputs of PDC Common Data Analysis pipeline? What do they mean?
PDC distributes several types of files, those submitted by original data submitters and those generated through the PDC common data analysis pipeline.

Mass Spectrometry Data Formats
RAW (Vendor) Format Mass spectrometry data uploaded by the data submitters as RAW or vendor format files corresponding to the mass spectrometers used to acquire the spectra.
mzML Format RAW format spectra in the HUPO Proteome Standards Initiative (PSI) External Link compliant mzML format..
Peptide-Spectrum Match (PSM) Data
RAW PSM Format The best peptide-spectrum matches (PSMs), from the first-level analysis of the PDC CDAP, for each tandem-mass spectrum against the peptide sequences from a reference protein sequence database (Uniprot) in tsv format.
mzIdentML PSM Format Raw PSMs in the PSI compliant mzIdentML format.
Protein Assembly
Protein identification and quantitation reports generated from the PSM data through a conservative gene-based generalized parsimony analysis. Peptides are associated with genes, rather than protein identifiers, and genes with at least two unshared peptide identifications are inferred. The resulting gene list is estimated to have a false-discovery rate of at most 1%. Several different output files are generated depending on the experiment type,
.summary.tsv - Protein identification summary report
.precursor_area.tsv - Label-free workflow protein quantitation report for relative quantitation by precursor peak area integration
.spectral_count.tsv - Label-free workflow protein quantitation report for relative quantitation by spectral counts
.itraq.tsv - iTRAQ workflow protein relative quantitation report
.tmt.tsv - TMT workflow protein relative quantitation report
.peptides.tsv - Identified peptide summary report
.phosphopeptide.tsv - Labelled workflow phosphopeptide relative quantitation report
.phosphosite.tsv - Labelled workflow phosphopeptide relative quantitation report
.glycopeptide.tsv - Labelled workflow N-linked glycopeptide relative quantitation report
.glycosite.tsv - Labelled workflow N-linked glycosite relative quantitation report
QC reports Quality control metrics computed by the CDAP, the report consists of summary statistics derived from all MS/MS spectra from the raw spectral datafiles.
Other metadata These are supplementary files provided by the original data submittres, these usually consists of,
  • Descriptive protocols
  • Clinical metadata
  • Other useful information
How do I cite the PDC?

To cite the resource, cite PDC url - https://pdc.cancer.gov
A PDC study can be cited with its PDC study identifier or url to the study. Ex: PDC000220 or https://pdc.cancer.gov/pdc/study/PDC000220

PDC may impose an embargo on certain datasets at the request of data submitters. In PDC, an embargo is the release of data on the PDC portal with the condition that users who access these datasets from the PDC portal cannot published or disseminated before a certain date. The embargo date is displayed on the study summary page of each dataset as applicable.

Can I use PDC data from NCI Cloud Resource to run my own analyses?

PDC data is located on Amazon Web Services S3 cloud buckets. All of the PDC data and metadata is accessible through APIs. There is an ongoing effort within the Cancer Research Data Commons for the Cloud Resources to run analysis pipelines on PDC data.
A tutorial on navigating the database is available from Seven Bridges Cancer Genomic Cloud (CGC) External Link, one of NCI’s Cloud Resources. Using the Seven Bridges “Data Cruncher,” and a specially designed PDC metadata file (PDC_metadata.json) that’s available in the public reference resources, users can extract data from the PDC. Other NCI Cloud Resources include the Broad Institute’s FireCloud and the Institute for Systems Biology’s Cancer Genomics Cloud.

Can I create a custom cohort to download?

PDC portal allows users to build cohorts by applying various clinical, biospecimen, experimental and file features as filters and export the selections as manifest files.

What data are not open-access?

Currently all of the data in PDC is open access.
In the near future, PDC is expected to host data submitters provided patient sample specific protein sequence databases. These databases are generated using the genomic and transcriptomic information from the patient sample and helps in identification of novel proteins, resulting from single nucleotide variants, splice variants and fusion genes. These files are currently designated as controlled access databases and would need dbGaP authorization for access.
